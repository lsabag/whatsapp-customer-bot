הבעיה

כל בעל חנות איקומרס מתמודד עם אותן שלוש בעיות:

שירות לקוחות ידני

נציגים עונים על אותן שאלות שוב ושוב. "כמה עולה המשלוח?" "מה מדיניות ההחזרות?" "יש את זה במידה L?" - שאלות שחוזרות עשרות פעמים ביום, גוזלות זמן ועולות כסף.

התוכן משתנה כל הזמן

מחירים מתעדכנים, מוצרים נכנסים ויוצאים, מבצעים מתחלפים. בוט רגיל שתכנתו אותו פעם אחת לא יודע את זה - הוא ממשיך לענות לפי מה שהיה נכון לפני חודשיים.

כל לקוח על פלטפורמה אחרת

אחד על WooCommerce, אחד על Shopify, אחד על מערכת מותאמת אישית. אי אפשר לבנות פתרון אחד ולמכור אותו לכולם - או שכן?

הפתרון - ארכיטקטורה

הפתרון שנבנה יחד מחולק לשכבות שעובדות יחד:

מקורות מידע

הבוט שואב מידע משלושה מקורות:

  • • סריקה אוטומטית של האתר
  • • ייבוא שיחות וואטסאפ קיימות שמלמדות אותו איך לקוחות אמיתיים שואלים
  • • מסמך הנחיות שמגדיר איך הבוט אמור להתנהג

Knowledge Graph

המוח האמיתי. RAG Engine רגיל זה כבר כנראה שייך לעבר.

במקום "לזרוק" טקסט לבוט, אנחנו ממפים קשרים: מוצר ↔ קטגוריה ↔ עונה ↔ מבצע, כי היום יותר ויותר עוברים להבנת הקשר עמוקה.

Vector Search

מנוע החיפוש החכם (ChromaDB) שעובד בשילוב עם הגרף. לא סתם מחפש מילות מפתח, אלא מחפש את המידע הקשור ביותר על בסיס משמעות.

למה צריך אותו אם יש גרף? הגרף הוא המפה, וה-Vector Search הוא ה-GPS: מבין את כוונת השאלה ומנחית אותנו בנקודה המדויקת בגרף שממנה נתחיל לטייל.

API חי

לשאלות אישיות כמו "איפה ההזמנה שלי?" הבוט מתחבר ישירות ל-WooCommerce ומושך נתונים בזמן אמת.

Router חכם

רכיב שמחליט אוטומטית לאן לפנות: שאלה כללית על מוצרים ומדיניות - RAG. שאלה אישית על הזמנה - API.

Knowledge Graph - איך זה עובד?

RAG Engine זה כבר כנראה שייך לעבר, והיום יותר ויותר הולכים ל-Knowledge Graph.

מיפוי קשרים במקום מילות מפתח

במקום לסרוק טקסט ארוך בתקווה למצוא התאמה, גרף הידע ממפה את החנות כמו רשת חכמה של מושגים (Entities) וקשרים ביניהם.

ישויות: מוצרים, קטגוריות, עונות, לקוחות.
קשרים: "שייך ל-", "מתאים ל-", "מוזל ב-".
השתמש ב-Ctrl + גלילה כדי להתקרב

כך הבוט יכול להמליץ על "ג'קט ג'ינס" כשמבקשים בגד חורף במבצע, גם אם המילים "חורף" או "מבצע" לא כתובות בו בכלל!

RAG / בוט רגיל

לקוח שואל "מה מתאים לחורף?". הבוט מחפש את המילה "חורף" בכל המסמכים ומחזיר את מה שמצא. אם המילה לא מופיעה בתיאור הג'קט, הוא לא מוצא אותו, גם אם זה בדיוק מה שהלקוח צריך.

Knowledge Graph

הבוט שלנו מטייל על "הקווים" בגרף. הוא מתחיל מ"חורף", רואה ש"ג'קטים" מחוברים אליו, ומושך את "ג'קט ג'ינס" יחד עם מבצע ה-30%. הכל תוך אלפיות שנייה ובהקשר מושלם.

שלבי הבנייה

1

אתר HTML

בניתי חנות בגדים מלאה בעברית בקובץ HTML אחד. מוצרים עם תמונות, עמוד FAQ, מדיניות משלוחים, מבצעים. זה משמש גם כבסיס לדמו וגם כמקור לסריקה.

2

Knowledge Graph

ממפים את כל הקשרים בחנות: מוצר ↔ קטגוריה ↔ עונה ↔ מבצע. עוברים משיטת חיפוש ישנה של RAG למבנה ידע חכם שמבין הקשרים.

3

Vector Database

הקמת ChromaDB לוקאלי. כאן אנחנו הופכים את הידע והקשרים שיצרנו לוקטורים (Embeddings) המאפשרים חיפוש סמנטי מהיר וזול.

4

WooCommerce API

מחבר את הבוט לנתוני הזמנות חיים. סטטוס, מעקב, פרטי לקוח - בזמן אמת.

5

Router חכם

הלב של המערכת. מנתח כל שאלה ומחליט: RAG לשאלות כלליות, API לשאלות אישיות, הסלמה לנציג אנושי כשהבוט לא יודע.

6

ממשק + דמו

Streamlit מספק צ'אט מלא בעברית, RTL, עם היסטוריית שיחה וכרטיסי הזמנה. מוכן להצגה.

הדמו החי (התוצאה)

אחרי שבנינו הכל - בואו נראה את זה עובד. פותחים שני חלונות: בשמאל חנות אמיתית, בימין הבוט שלנו.

תרחיש 1 - שאלה על מוצר

"כמה עולה ג'קט עור ויש אותו בחום?" - הבוט עונה מיד עם מחיר מדויק וצבעים זמינים.

תרחיש 2 - סטטוס הזמנה חי

"מה הסטטוס של הזמנה 1050?" - הבוט שולף את פרטי ההזמנה בזמן אמת.

תרחיש 3 - עדכון תוכן

מעדכנים מחיר מוצר באתר תוך כדי ההצגה, מריצים סריקה מחדש - הבוט כבר יודע את המחיר החדש. זה הרגע שגורם לאנשים לומר "וואו".

תרחיש 4 - מה הבוט לא יודע

"יש הנחה לחיילים?" - הבוט אומר שאין לו מידע על כך ומציע להעביר לנציג אנושי. זה חשוב להראות - בוט שיודע מתי הוא לא יודע הוא בוט אמין.

פשטות בהקמה

עצמאות מלאה

כל לקוח מקבל תיקייה עצמאית עם ה-config שלו, מסמכי הידע שלו, ה-DB הלוקאלי שלו, ה-Graph שלו. אין תלות בין לקוחות, אין שרת מרכזי שיכול לקרוס.

הפתרון שלנו

$10-30 לחודש

רק OpenAI API לפי שימוש. אין עלות DB, אין עלות שרת אם רץ לוקאלית.

פתרונות חיצוניים (Intercom, Gorgias)

$200-500 לחודש

אנגלית בלבד, לא מותאם לעברית ולשוק הישראלי.

סיכום

מה בנינו בפועל:

  • אתר חנות מלא בעברית - בסיס לסריקה ולדמו
  • Knowledge Graph - הבנת קשרים בין מוצרים, קטגוריות ומבצעים
  • RAG Engine לוקאלי - ללא עלויות DB חיצוני
  • חיבור ל-WooCommerce API - נתוני הזמנות בזמן אמת
  • Template מודולרי - לקוח חדש = תיקייה + config
